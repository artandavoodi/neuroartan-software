# ICOS UX Asset Registry

## Purpose
Defines all static assets used across ICOS UX including images, icons, media resources, and system visuals.

## Core Principle
All assets must be centrally registered. No untracked or ad-hoc assets are allowed.

## Asset Rules

### 1. Centralization Rule
- All assets must be registered in this system
- No external or untracked asset usage permitted

### 2. Reusability Rule
- Assets must be reusable across multiple UI modules
- No screen-specific asset duplication allowed

### 3. Consistency Rule
- Same asset must render identically across all contexts

### 4. Optimization Rule
- Assets must be optimized for performance and minimal redundancy

## Asset Categories
- Icons
- Images
- Illustrations
- UI Backgrounds
- System Graphics

## System Binding
This registry binds to:
- Design System Core
- Layout Engine Core
- Components Registry
- Screens Index

## Constraints
- No unmanaged asset usage
- No duplicate asset definitions
- No inline asset embedding outside registry control

## Status
Active and enforced across UX architecture layer
