# ICOS Context Compression Doctrine

## Purpose

Defines how ICOS must compress context while preserving essential meaning, structure, and execution-critical information.

---

## Core Principle

Compression reduces size, not meaning.

Signal must remain intact.

---

## Compression Rule

When compression is required:

- remove redundancy
- condense verbose sections
- preserve key logic

---

## Preservation Rule

Compressed context must retain:

- execution instructions
- dependencies
- references
- structural meaning

---

## Reduction Hierarchy

Compression must remove in order:

```text
1. Duplicate content
2. Non-essential descriptions
3. Low-priority context
4. Historical data
```

Critical data must never be removed.

---

## Determinism Rule

Given identical input:

- compression output must be identical

---

## Structural Integrity

Compression must preserve:

- hierarchy
- relationships
- execution flow

---

## Loss Constraint

Compression must not:

- remove required dependencies
- break references
- distort meaning

---

## Failure Handling

If compression risks loss of critical data:

- do not compress
- request scope reduction

---

## Prohibited Behavior

ICOS must never:

- over-compress critical logic
- distort meaning
- remove dependencies

---

## Enforcement

- incorrect compression is a failure
- loss of critical data is invalid
- compression must be reversible in meaning

---

## Injection Rule

This doctrine must be included in every prompt before inference.
