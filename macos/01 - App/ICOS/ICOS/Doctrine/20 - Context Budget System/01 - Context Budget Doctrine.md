# ICOS Context Budget Doctrine

## Purpose

Defines how ICOS must allocate, manage, and optimize context usage to ensure efficient, deterministic, and complete execution within system limits.

---

## Core Principle

Context is finite.

Only the highest-value information may be included.

---

## Budget Rule

Context must:

- stay within token limits
- prioritize essential information
- exclude redundant data

---

## Priority Hierarchy

Context must be ordered by importance:

```text
1. Active Execution File
2. Direct Dependencies
3. Related Doctrines
4. Supporting Context
5. Historical / Low Priority Data
```

Higher priority must always be preserved.

---

## Allocation Rule

Context must allocate tokens to:

- instructions
- required references
- execution state

No unnecessary allocation allowed.

---

## Reduction Rule

When context exceeds budget:

- compress lower-priority data
- remove non-essential content

Critical information must never be removed.

---

## Determinism Rule

Given identical input and context:

- context selection must be identical

---

## Efficiency Rule

Context must:

- minimize redundancy
- avoid repetition
- maintain clarity

---

## Dependency Rule

All required dependencies must remain in context.

Missing dependency → invalid execution.

---

## Failure Handling

If context cannot fit within budget:

- halt execution
- request scope reduction

---

## Prohibited Behavior

ICOS must never:

- overload context
- include irrelevant data
- drop critical information silently

---

## Enforcement

- context overflow is a failure
- incorrect prioritization is invalid
- system must enforce strict context discipline

---

## Injection Rule

This doctrine must be included in every prompt before inference.
