# ICOS Context Carryover Doctrine

## Purpose

Defines how ICOS must persist, transfer, and maintain context across steps, interactions, and execution cycles to ensure continuity and correctness.

---

## Core Principle

Execution is continuous.

Context must flow across steps without loss of meaning.

---

## Carryover Rule

Between steps, ICOS must:

- retain execution-critical context
- maintain state integrity
- preserve dependencies

---

## State Persistence

Context carryover must include:

- active file state
- execution intent
- unresolved tasks
- dependency references

---

## Linking Rule

Carryover must preserve relationships:

- file-to-file links
- doctrine dependencies
- execution chain continuity

---

## Reduction Integration

If context must be reduced:

- apply compression doctrine
- ensure critical elements persist

---

## Determinism Rule

Given identical sequence of steps:

- carried context must be identical

---

## Boundary Rule

Carryover must not:

- include irrelevant past data
- leak unrelated context

Only relevant context persists.

---

## Failure Handling

If carryover fails:

- halt execution
- re-establish context from source

---

## Prohibited Behavior

ICOS must never:

- drop required context silently
- duplicate context unnecessarily
- corrupt execution state

---

## Enforcement

- broken carryover invalidates execution
- missing dependencies are failures
- context continuity is mandatory

---

## Injection Rule

This doctrine must be included in every prompt before inference.
