# ICOS No Overlay Doctrine

## Purpose

Defines the strict prohibition of overlay-based fixes, workarounds, or compensating layers. All corrections must be applied at the root cause.

---

## Core Principle

Fix the source.

Never patch the symptom.

---

## Definition of Overlay

Overlay is any solution that:

- adds a secondary layer to mask a problem
- compensates without resolving root cause
- duplicates logic to bypass failure

---

## Prohibition Rule

ICOS must never:

- add workaround layers
- stack fixes on top of broken logic
- preserve faulty structure with patches
- override global design tokens locally
- add compensating visual layers to hide token drift
- use feature-specific styling to fight system-level visual ownership

---

## Root Cause Rule

All fixes must:

- identify true origin of issue
- modify the source of failure
- remove need for compensation
- identify the true token owner when the issue is visual
- promote repeated visual values into named token ownership
- verify the correction with build and scan rather than visual assumption

---

## Detection Rule

A solution is invalid if it:

- introduces redundancy
- increases complexity without necessity
- avoids modifying the original failure point

---

## Token Overlay Rule

Visual drift must be corrected through token ownership.

A visual correction is invalid if it:

- patches one feature while leaving the system token unchanged
- creates a local duplicate of a global visual rule
- uses opacity, radius, spacing, frame, font, or material values directly inside a feature surface when a token owner exists
- preserves a broken token relationship by adding a new visual wrapper

Corrective path:

1. identify the canonical owner
2. update or create the named token
3. replace local literals with token references
4. build immediately
5. run a targeted scan
6. classify remaining hits as unresolved or accepted-native

---

## Determinism Rule

Root fixes must:

- simplify system state
- produce consistent behavior

---

## Failure Handling

If root cause cannot be identified:

- halt execution
- do not apply temporary fix

---

## System Integrity

Overlay solutions degrade:

- maintainability
- predictability
- correctness

Therefore they are prohibited.

---

## Enforcement

- overlay attempts must be rejected
- non-root fixes are invalid
- system must enforce root-level correction
- token drift must be corrected at the token owner
- visual changes must be token-first, build-verified, and scan-classified
- native platform exceptions must be explicitly recognized rather than patched away

---

## Injection Rule

This doctrine must be included in every prompt before inference.
