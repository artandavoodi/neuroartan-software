# ICOS Ownership Detection Doctrine

## Purpose

Defines how ICOS must detect and resolve the single authoritative owner of any logic, structure, or content before applying changes.

---

## Core Principle

Every element has one owner.

All modifications must target the owner.

---

## Ownership Rule

Before any modification, ICOS must:

- identify the source of truth
- confirm ownership of the logic or structure
- target only the owning layer

---

## Ownership Types

Ownership may exist in:

- a single file
- a canonical doctrine
- a runtime component
- a shared system layer

Only one owner is authoritative.

---

## Detection Process

```text
1. Identify affected element
2. Trace origin of definition
3. Locate canonical source
4. Confirm no higher-level owner exists
5. Select single owner
```

---

## Conflict Rule

If multiple possible owners exist:

- do not modify
- perform deeper scan
- resolve to single authority

---

## Hierarchy Rule

Ownership priority:

```text
1. Core System Layer
2. Doctrine Layer
3. Local File Layer
4. Derived / Output Layer
```

Higher layer overrides lower.

---

## Edit Rule

Edits must:

- occur only in the owner
- not duplicate logic elsewhere
- not override from non-owner layers

---

## Determinism Rule

Given identical structure:

- ownership detection must produce identical owner

---

## Failure Handling

If ownership cannot be determined:

- halt execution
- do not apply modification

---

## Prohibited Behavior

ICOS must never:

- modify non-owner layers
- duplicate logic across files
- override behavior from outside the owner

---

## Enforcement

- incorrect ownership targeting is a failure
- duplicate ownership is invalid
- system must maintain single source of truth

---

## Injection Rule

This doctrine must be included in every prompt before inference.
