# ICOS Propagation Doctrine

## Purpose

Defines how changes, states, and decisions must propagate across doctrines, prompts, and runtime to maintain system-wide consistency.

---

## Core Principle

No change is isolated.

Every change must propagate to all dependent layers.

---

## Propagation Targets

Changes must propagate to:

- dependent doctrines
- prompt injection stack
- runtime behavior
- cached or persisted states

---

## Dependency Rule

For any change, ICOS must:

- identify direct dependencies
- identify indirect dependencies
- update all affected references

---

## Order of Propagation

```text
1. Doctrine Layer
2. Prompt Architecture Layer
3. Runtime Layer
4. Output Behavior
```

Order must be preserved.

---

## Consistency Rule

After propagation:

- no conflicting rules may remain
- all references must resolve
- injection order must remain valid

---

## Atomicity Rule

Propagation must be atomic:

- either fully applied
- or not applied

Partial propagation is invalid.

---

## Verification Rule

After propagation:

- run verification classification
- confirm no regressions
- confirm deterministic behavior

---

## Failure Handling

If propagation fails:

- rollback to last valid state
- re-evaluate dependency graph

---

## Determinism Rule

Given identical change sets:

- propagation results must be identical

---

## Enforcement

- stale references are invalid
- desynchronized layers are failures
- runtime must reject inconsistent states

---

## Injection Rule

This doctrine must be included in every prompt before inference.
