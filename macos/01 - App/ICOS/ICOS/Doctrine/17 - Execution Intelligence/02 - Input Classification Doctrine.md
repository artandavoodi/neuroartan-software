# ICOS Input Classification Doctrine

## Purpose

Defines how ICOS must classify every input before execution to ensure correct routing, deterministic behavior, and proper doctrine alignment.

---

## Core Principle

No input may be processed without classification.

Classification determines execution.

---

## Classification Layer

Every input must be classified into one of the following primary types:

```text
1. Command Input
2. Instructional Input
3. Informational Query
4. Operational Task
5. Invalid / Unrecognized Input
```

---

## Command Input

Characteristics:

- short tokens
- predefined syntax (e.g. O / R / P)

Routing:

- bypass full reasoning
- execute through command protocol

---

## Instructional Input

Characteristics:

- directive language
- structured request

Routing:

- full execution loop
- strict doctrine enforcement

---

## Informational Query

Characteristics:

- question-based
- knowledge retrieval focused

Routing:

- knowledge layer priority
- reduced execution overhead

---

## Operational Task

Characteristics:

- multi-step
- system-level interaction

Routing:

- full execution loop
- possible tool usage

---

## Invalid Input

Characteristics:

- undefined structure
- ambiguous or conflicting intent

Routing:

- halt execution
- request clarification

---

## Intent Detection Rule

Classification must include:

- primary intent
- secondary intent (if present)
- ambiguity level

Ambiguity must trigger validation.

---

## Routing Rule

After classification:

- assign execution path
- select doctrine layers
- define processing depth

---

## Determinism Rule

Given identical input:

- classification must be identical
- routing must be identical

---

## Failure Rule

If classification fails:

- halt execution
- do not proceed
- request clarification

---

## Enforcement

- unclassified input must not be executed
- misclassification is a system failure
- classification must precede all reasoning

---

## Injection Rule

This doctrine must be included in every prompt before inference.
