# ICOS Memory Retrieval Rules Doctrine

## Purpose

Defines how memory is retrieved, filtered, prioritized, and injected into runtime without violating system authority or product definition.

---

## Retrieval Principle

Memory retrieval must be:

- relevant
- minimal
- structured
- non-authoritative

Memory supports execution.

Memory does not define truth.

---

## Retrieval Conditions

Memory may be retrieved only when:

- directly relevant to the task
- required for continuity
- necessary for accuracy

Memory must not be retrieved when:

- unrelated to the current task
- conflicting with higher-order doctrine
- excessive or redundant

---

## Retrieval Priority

When multiple memories exist:

1. Most recent
2. Most relevant
3. Highest confidence

Older or lower-confidence memory must be ignored if conflict exists.

---

## Authority Rule

Memory is always lower authority than:

- Identity
- Product Definition
- Model Separation
- Runtime Contract

Memory must never override system doctrine.

---

## Injection Rule

Memory is injected after:

- core doctrine
- runtime contract
- prompt architecture

Memory is injected before:

- task execution

---

## Formatting Rule

Memory must be injected in structured format:

```text
[MEMORY]
key: value
[/MEMORY]
```

No unstructured memory insertion is allowed.

---

## Conflict Handling

If memory conflicts with doctrine:

- ignore memory
- preserve doctrine

If memory conflicts with memory:

- use highest priority memory
- discard others

---

## Performance Constraint

Memory retrieval must be:

- fast
- bounded
- efficient

Do not load full memory history.

Retrieve only necessary segments.

---

## Enforcement

- Memory retrieval must pass relevance validation
- Invalid memory must be ignored
- System must continue without degraded behavior

---

## Injection Rule

This doctrine must be included in every prompt before inference.
