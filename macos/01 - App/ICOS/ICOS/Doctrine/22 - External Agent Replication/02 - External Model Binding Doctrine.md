# ICOS External Model Binding Doctrine

## Purpose

Defines how ICOS must bind to external models while preserving deterministic behavior, doctrinal integrity, and execution consistency.

---

## Core Principle

The model is not the system.

ICOS controls the model, not the inverse.

---

## Binding Rule

External models must be:

- wrapped in ICOS execution framework
- governed by ICOS doctrines
- restricted to defined interfaces

---

## Interface Layer

All interaction with external models must pass through:

```text
Input → ICOS Layer → Model → ICOS Layer → Output
```

No direct model access allowed.

---

## Control Rule

ICOS must control:

- prompt structure
- context injection
- execution flow

External models must not alter system logic.

---

## Determinism Rule

Given identical input and context:

- model output must be normalized
- final ICOS output must be identical

---

## Normalization Rule

Model outputs must be:

- validated
- classified
- normalized into ICOS format

Raw model output must never be exposed.

---

## Constraint Rule

External models must respect:

- execution loop
- input classification
- verification classification
- no overlay doctrine

---

## Variance Handling

If model output varies:

- normalize through verification
- enforce deterministic structure

---

## Failure Handling

If model behavior deviates:

- treat as external failure
- halt execution if required

---

## Prohibited Behavior

ICOS must never:

- delegate control to model
- trust model output blindly
- expose raw model responses

---

## Enforcement

- model must be fully governed
- unbound model usage is invalid
- ICOS remains authoritative layer

---

## Injection Rule

This doctrine must be included in every prompt before inference.
