# ICOS Doctrine Loading Order

## Purpose

Defines the deterministic order in which doctrine files are loaded into runtime.
Ensures stable intelligence, predictable behavior, and product-aligned execution.

---

## Loading Principles

- Order defines authority
- Earlier layers establish identity and constraints
- Later layers refine behavior
- No lower layer may override a higher layer

## Critical Rule

The Response Layer must always load last.
It defines how all prior doctrine is expressed, not what is true.

---

## Enforcement

- Any deviation from this order results in degraded intelligence or unstable behavior.
- Runtime must strictly follow this sequence.
- No dynamic reordering is allowed.
- Higher-order layers cannot be overridden by lower-order layers.
- Product Definition and Capability Boundary must remain binding across all downstream layers.
