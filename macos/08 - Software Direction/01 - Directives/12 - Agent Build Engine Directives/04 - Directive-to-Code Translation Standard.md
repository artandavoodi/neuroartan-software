## PURPOSE

Define how ICOS directives are translated into executable code with full architectural alignment.

---

## CORE POSITION

Directives are specifications.

Code is execution.

Translation must be structural, not interpretive.

---

## TRANSLATION PIPELINE

The agent must follow a fixed pipeline:

1. directive parsing
2. structural decomposition
3. system mapping
4. dependency resolution
5. plan generation
6. code generation
7. validation

No stage may be skipped.

---

## DIRECTIVE PARSING

Extract:

- intent
- affected layers
- constraints
- success conditions

Ambiguity must be resolved structurally.

---

## STRUCTURAL DECOMPOSITION

Break directive into:

- components
- data structures
- services
- interfaces
- state transitions

---

## SYSTEM MAPPING

Map components to ICOS layers:

- runtime
- orchestration
- memory
- profile
- interface
- voice
- continuity

---

## DEPENDENCY RESOLUTION

Identify:

- prerequisites
- existing modules
- required integrations
- ordering constraints

---

## PLAN GENERATION

Produce an execution plan:

- ordered steps
- target files/modules
- data models
- APIs/contracts
- test cases

Plan must be deterministic.

---

## CODE GENERATION RULES

Code must be:

- modular
- layered
- token-consistent
- parity-compliant with website
- free of hardcoded logic

No UI-first implementations.

---

## STATE & DATA MODELS

Define explicit models for:

- sessions
- models (registry)
- profiles
- permissions
- memory entries

Schemas must be versioned and validated.

---

## ROUTING & RUNTIME

Ensure:

- model selection binds to real engine
- session carries model state
- runtime exposes lifecycle

---

## VALIDATION

Validate against:

- directive intent
- architecture compliance
- product identity
- permission rules
- parity rules

Include automated checks where possible.

---

## TESTING

Generate tests:

- unit tests for components
- integration tests for flows
- permission tests
- routing tests

No untested code paths.

---

## NO OVERLAY RULE

Fix at source.

No compensating layers.

---

## FAILURE HANDLING

On failure:

- halt generation
- report root cause
- regenerate plan

---

## OUTPUT ARTIFACTS

Each translation must output:

- plan
- code
- tests
- validation report

---

## SUCCESS CONDITION

Directive results in deployable, validated, system-aligned code.
