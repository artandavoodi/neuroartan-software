## PURPOSE

Define how the ICOS internal agent ingests context and maintains full product awareness during execution.

---

## CORE POSITION

The agent must operate with full-system awareness.

Local context is insufficient.

---

## CONTEXT INTAKE MODEL

The agent must continuously ingest:

- active directive
- related directives
- product definition
- software direction documents
- system state

---

## CONTEXT HIERARCHY

Context must be prioritized in this order:

1. Product Identity
2. Core Directives
3. System Directives
4. Local Directive
5. Runtime State

No inversion allowed.

---

## PRODUCT AWARENESS

The agent must always maintain awareness of:

- ICOS identity
- system layers
- architecture rules
- directive relationships

---

## GLOBAL AWARENESS RULE

The agent must not operate on:

- isolated file context
- partial system view

All actions must consider full system impact.

---

## CONTEXT RESOLUTION

The agent must resolve conflicts by:

- prioritizing higher-level directives
- preserving product identity
- maintaining system integrity

---

## DIRECTIVE LINKING

The agent must:

- identify related directives
- integrate their constraints
- ensure alignment across documents

---

## CONTEXT VALIDATION

Before execution, the agent must validate:

- context completeness
- directive alignment
- system consistency

---

## CONTEXT FAILURE HANDLING

If context is incomplete:

- pause execution
- request missing context or scan

No blind execution.

---

## NO LOCAL OPTIMIZATION

The agent must not:

- optimize locally while breaking global structure
- prioritize speed over system integrity

---

## SYSTEM MEMORY OF CONTEXT

The agent must retain awareness of:

- previous directives
- system evolution
- structural decisions

---

## SOFTWARE RULE

Context awareness must be treated as a core capability of the agent.

Not optional.

---

## FAILURE CONDITION

Agent acts on partial context and creates inconsistent system behavior.

---

## SUCCESS CONDITION

Agent executes with full awareness of product, system, and directive relationships.
