## PURPOSE

Define the canonical search-first architecture of ICOS as the primary entry point into the system.

---

## CORE POSITION

ICOS is search-first.

ICOS is not prompt-box-first.

Discovery precedes interaction.

---

## ENTRY MODEL

User must be able to:

- search immediately
- view models
- select a model
- interact with the real engine

No friction before discovery.

---

## SEARCH ARCHITECTURE

Search must operate over the Model Registry.

Search must not be a UI filter.

Search must be a system layer.

---

## SEARCH INPUT TYPES

System must support:

- keyword search
- semantic search
- identity-based search
- intent-based search

---

## SEARCH OUTPUT

Search must return governed models.

Each result must expose:

- model identity
- owner
- verification state
- reliability signal
- availability
- interaction permission

---

## RANKING RULE

Search ranking must prioritize:

- verified models
- higher reliability
- stronger training depth
- permitted access

No manipulation.

---

## PERMISSION FILTERING

Search results must respect:

- visibility state
- permission state
- entitlement

User must never see inaccessible models incorrectly.

---

## MODEL SELECTION RULE

Selecting a model must:

- bind to runtime
- change execution engine
- update session state

No UI-only selection.

---

## DISCOVERY STATES

System must support:

- public searchable
- public non-searchable
- searchable non-interactable
- interactable

These must remain separate.

---

## TRUST SIGNALS

Search must expose:

- verification badges
- reliability indicators
- training maturity

These must be real.

---

## GUEST ACCESS RULE

Guest users must:

- access limited models
- remain permission-bound

No unrestricted interaction.

---

## SYSTEM INTEGRATION

Search must integrate with:

- model registry
- permission system
- dashboard signals
- profile system

---

## SOFTWARE RULE

Search must exist as a core system layer.

No UI-only filtering.

No duplicated logic.

---

## SUCCESS CONDITION

User enters ICOS through discovery and interacts with real governed models.
