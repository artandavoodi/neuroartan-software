## PURPOSE

Define the canonical architecture for ICOS profile and voice systems as core identity infrastructure.

---

## CORE POSITION

Profile is the intelligence surface.

Voice is identity infrastructure.

Neither are UI features.

---

## PROFILE ARCHITECTURE

Profile must act as a structured cognitive entity.

System must support:

- explicit user data
- reflection history
- thought-pattern recognition
- training state
- reliability signals

---

## PROFILE LAYERS

System must distinguish:

### 01 · Core Profile

- internal
- private
- authoritative identity layer

### 02 · Public Profile

- permission-bound
- visibility-controlled

### 03 · Interaction Surface

- used for session engagement
- context-aware

---

## VOICE SYSTEM POSITION

Voice is not cosmetic.

Voice represents trained continuity and identity expression.

---

## VOICE TRAINING REQUIREMENT

System must support:

- voice sample ingestion
- pattern extraction
- behavioral shaping
- evaluation of fidelity

---

## TRAINING INPUT TYPES

System must support:

- direct user input
- uploaded sources
- authorized external accounts
- reflection-derived signals

---

## SOURCE GOVERNANCE

All training input must preserve:

- consent
- provenance
- permission scope
- revocation ability

---

## TRAINING VS CONTEXT RULE

System must strictly separate:

- contextual usage
- training usage

No implicit upgrade allowed.

---

## RELIABILITY SIGNALS

Profile must expose:

- voice readiness
- training depth
- source coverage
- thought-pattern maturity
- reliability state

These must be real, not simulated.

---

## VISIBILITY RULE

Profile must support:

- private state
- public state
- searchable state
- interactable state
- legacy-enabled state

Each independently governed.

---

## LEGACY CONTINUITY RULE

Voice-based legacy interaction must require:

- explicit authorization
- verified identity
- permission-bound exposure

No default activation.

---

## IDENTITY PROTECTION

System must prevent:

- impersonation
- identity leakage
- unauthorized voice exposure

---

## SYSTEM INTEGRATION

Profile and voice must connect to:

- model registry
- session system
- memory system
- dashboard
- search and discovery

---

## SOFTWARE RULE

Profile and voice must exist as core system layers.

No UI-only representation.

No duplicated logic.

---

## SUCCESS CONDITION

System produces stable identity, governed voice, and traceable training progression.
