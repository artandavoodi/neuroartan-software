## PURPOSE

Define the ICOS unified workspace as the single interface where all cognition, execution, and creation occur.

---

## CORE POSITION

There is no separation between tool and output.

Workspace is the system.

---

## WORKSPACE STRUCTURE

ICOS must operate through a unified multi-panel interface:

### LEFT PANEL · Navigation

- projects
- documents
- models
- sessions

---

### CENTER PANEL · Active Work Surface

- document editing
- code writing
- structured creation

This is the primary interaction layer.

---

### RIGHT PANEL · Execution & Context

- agent execution logs
- model state
- session state
- runtime outputs

---

### PREVIEW PANEL

- rendered UI
- app previews
- document output

User can interact directly with preview.

---

## INTERACTION MODEL

User must be able to:

- type instructions
- edit documents directly
- comment on outputs
- trigger execution

All interactions are continuous and stateful.

---

## DIRECT MANIPULATION

User must:

- comment on preview elements
- highlight sections for refinement
- guide agent behavior in context

No detached prompting model.

---

## STATE AWARENESS

Workspace must always display:

- active model
- execution state
- task progress
- errors and logs

---

## SESSION INTEGRATION

Workspace must bind:

- documents
- sessions
- model state

All activity must persist.

---

## MULTI-TASK VIEW

Workspace must support:

- multiple active tasks
- switching between executions
- parallel task visibility

---

## FILE SYSTEM INTEGRATION

Workspace must:

- reflect actual file structure
- allow direct file editing
- sync with local and cloud storage

---

## EXECUTION VISIBILITY

All agent actions must be:

- visible
- traceable
- interruptible

---

## NO FRAGMENTATION RULE

The system must not:

- separate chat from execution
- separate files from interface

Everything lives inside the workspace.

---

## FAILURE CONDITION

Workspace behaves like a chat UI or disconnected tool panels.

---

## SUCCESS CONDITION

Workspace becomes a continuous environment where thinking, building, and execution occur simultaneously.
