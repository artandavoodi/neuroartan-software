## PURPOSE

Establish foundational user practices for operating ICOS effectively, covering prompting, permissions, model usage, organization, and parallel workflows.

---

## CORE POSITION

Prompting is orchestration.

The user directs; ICOS executes.

---

## BASICS · PROMPTING

Effective prompts must:

- define intent
- specify output type
- include constraints
- reference context (files, project, model)

Preferred structure:

- objective
- scope
- constraints
- deliverable

---

## DIRECT MANIPULATION

Prefer in-context control:

- edit documents directly
- comment on previews
- highlight sections for change

Avoid detached instructions when context exists.

---

## PERMISSIONS & SAFETY

Users must:

- grant explicit permissions per task
- define scope (files, apps, browser)
- review actions before execution

High-impact actions require confirmation.

---

## MODEL SELECTION

Choose models by task:

- reasoning → high-cognition models
- coding → code-optimized models
- media/design → multimodal models

Routing should match task requirements.

---

## ORGANIZATION

Maintain structure:

- use projects and folders
- keep documents versioned
- name artifacts clearly

All outputs persist in workspace.

---

## WORKFLOW PATTERN

Standard loop:

1. define intent
2. generate draft
3. refine in-place
4. validate
5. build/deploy

Repeat as needed.

---

## MULTITASKING

Operate in parallel:

- run multiple tasks simultaneously
- monitor progress in workspace
- reprioritize as needed

Avoid serial execution for independent tasks.

---

## AUTOMATIONS

Use automations to:

- chain repetitive steps
- standardize outputs
- reduce manual work

Store and reuse workflows.

---

## COMPUTER USE

When enabling device control:

- define exact targets
- keep scope minimal
- monitor execution

Always maintain visibility and control.

---

## COMMON PATTERNS

- Build app + deck + video in parallel
- Draft → review → finalize documents
- Import data → transform → export

---

## FAILURE CONDITION

User operates ICOS as a simple chat tool with vague prompts and no structure.

---

## SUCCESS CONDITION

User orchestrates ICOS to execute structured, parallel workflows with precise control and high-quality outputs.
