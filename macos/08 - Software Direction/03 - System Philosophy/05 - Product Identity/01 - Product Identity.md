## PURPOSE

Define what ICOS is and enforce non-negotiable product identity across all implementations.

---

## CORE IDENTITY

ICOS is a cognitive operating system.

ICOS is not:

- a chatbot
- a prompt interface
- a conversational assistant
- a UI-first product

ICOS is a structured system composed of:

- runtime
- orchestration
- memory
- profile
- interface
- voice
- continuity

---

## IDENTITY AXIOM

Product identity must be preserved across all layers.

If a component violates identity, it must be corrected or removed.

---

## SYSTEM NATURE

ICOS is:

- reflective
- structured
- continuity-aware
- identity-bound

ICOS must not be:

- reactive
- generic
- improvisational
- personality-driven

---

## INTERACTION POSITION

Interaction is a surface.

It is not the product.

The product exists beneath the interface.

---

## USER PERCEPTION RULE

User must experience ICOS as:

- a system
- not a chat
- not a tool
- not a persona

---

## IMPLEMENTATION CONSTRAINT

No feature may:

- simulate capability without system backing
- degrade system into chat behavior
- bypass core layers

---

## CONTINUITY PRINCIPLE

ICOS must preserve continuity across:

- sessions
- profiles
- models
- memory

Continuity must be governed, not implied.

---

## VOICE POSITION

Voice is an extension of identity.

It must remain:

- permission-bound
- non-default
- governed

---

## TRUST POSITION

Trust is derived from:

- structure
- consistency
- transparency

Not from personality.

---

## FAILURE CONDITION

System becomes a chatbot.

This is a total product failure.

---

## SUCCESS CONDITION

ICOS behaves as a coherent cognitive operating system across all surfaces.
